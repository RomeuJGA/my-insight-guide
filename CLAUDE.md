# Um Ävatar — Instruções para Claude Code

## Regra fundamental

**Antes de qualquer alteração ao código, é obrigatório:**

1. Apresentar um plano detalhado do que vai ser feito
2. Aguardar aprovação explícita do utilizador
3. Só então executar as alterações

Esta regra aplica-se a todas as alterações — novas funcionalidades, correções de bugs, ajustes de texto, estilos, migrações de base de dados, e deploys.

Exceção permitida: erros de compilação ou TypeScript introduzidos pela própria alteração aprovada podem ser corrigidos sem novo plano.
